### 12 креативних практичних завдань (від простого → складного)

---

**1. Розвідка структури БД**
Знайти:

- кількість записів у кожній таблиці
- мінімальну та максимальну дату звітів
- список підрозділів, які мають звіти

Результат: 3–5 простих SELECT.

---

**2. Базовий фільтр по часу**
Знайти всі звіти:

- за останні 7 днів
- за останній місяць
- тільки типу SITREP

Результат: SELECT + WHERE + BETWEEN.

---

**3. Аналітика по підрозділах**
Порахувати:

- кількість звітів на підрозділ
- середню критичність (severity) на підрозділ

Результат: GROUP BY.

---

**4. Аналітика по напрямках**
Визначити:

- який напрям має найбільше звітів
- який має найбільшу середню severity

Результат: GROUP BY + ORDER BY.

---

**5. Робота з метриками (JOIN)**
Знайти:

- сумарні enemy_kia за підрозділами
- сумарні engagements за днями

Результат: JOIN report + report_item.

---

**6. Побудова “оперативного зрізу дня”**
Для кожного дня:

- кількість звітів
- середня severity
- сумарні engagements

Результат: GROUP BY date.

---

**7. Псевдо-рейтинги підрозділів**
Побудувати рейтинг підрозділів за:

- total engagements
- total enemy losses

Результат: ORDER BY + LIMIT.

---

**8. Віконні функції — базовий рівень**
Для кожного звіту показати:

- середню severity по підрозділу
- різницю між severity звіту та середньою

Результат:
window functions (_window functions_ /ˈwɪndoʊ ˈfʌŋkʃənz/)

---

**9. Віконні функції — тренд по часу**
Для кожного підрозділу:

- ковзне середнє severity за 7 днів
- накопичувальну суму engagements

Результат:
OVER + ORDER BY + ROWS BETWEEN.

---

**10. Виявлення аномалій**
Знайти звіти, де:

- engagements > середнього по підрозділу × 2
- або enemy_kia сильно відхиляється від середнього

Результат: підзапити + window avg/stddev.

---

**11. Побудова “оперативної картини” (міні BI-запит)**
Створити один запит, який показує:

- підрозділ
- день
- engagements
- enemy_kIA
- середню severity
- місце підрозділу у рейтингу дня

Результат: JOIN + GROUP BY + RANK().

---

**12. Рівень “штабний аналітик” — складна аналітика**
Побудувати модель:

- індекс інтенсивності боїв = engagements + enemy_kia × коефіцієнт
- тренд інтенсивності по підрозділах
- визначити ТОП-3 “найгарячіших” періодів

Результат:

- CTE
- window functions
- ranking
- складні обчислення
